## 🎯 技能概述

---

## 📋 通信模式

### 模式一：飞书消息回复

**触发条件：** 检测到飞书收到新消息（通过日志监控）

**行为：**
1. 检查消息内容
2. 判断是否需要回复
3. 主动调用 `message` 工具发送到飞书

**示例流程：**
```
1. 日志显示：feishu[default]: received message from ou_xxx
2. 开阳检测到消息
3. 判断消息类型（询问/指令/闲聊）
4. 调用 message.send 回复
```

**回复原则：**
- ✅ 询问类消息 → 立即回复
- ✅ 指令类消息 → 确认后回复
- ✅ 闲聊类消息 → 选择性回复
- ⚠️ 重复消息 → 去重处理

---

### 模式二：重要事项主动汇报

**触发条件：**
- 开发任务完成阶段性目标
- 系统出现异常/错误
- 需要用户决策/协调
- 每日工作结束

**汇报内容：**
1. 任务进度
2. 遇到的问题
3. 需要的支持
4. 下一步计划

**汇报格式：**
```
## 📊 开阳汇报 - [主题]

**时间：** YYYY-MM-DD HH:mm
**进度：** XX%

### ✅ 已完成
- ...

### 🚧 进行中
- ...

### 📋 需要你协调的
- ...

### 🎯 下一步
- ...
```

---

### 模式三：深度开发对话

**使用场景：**
- 代码开发
- 系统设计
- 复杂问题讨论
- 文件编辑操作

**工作位置：** webchat 控制台

**原因：**
- webchat 支持完整工具链
- 可以实时查看日志
- 方便文件操作和调试
- 对话历史完整

---

## 🔧 实现方式

### 1. 日志监控（被动接收）

```bash
# 监控飞书消息日志
tail -f /tmp/openclaw/openclaw-*.log | grep "feishu.*received message"
```

**检测逻辑：**
- 定期轮询日志（每 30 秒）
- 检测新消息到达
- 提取发送者和内容

### 2. 主动发送

```python
# 使用 message 工具发送
message.send(
    channel="feishu",
    target="ou_xxx",
    message="内容"
)
```

### 3. 心跳检查

**频率：** 每 2-4 小时

**内容：**
- 系统状态
- 任务进度
- 是否有问题

## 🎯 最佳实践

### ✅ 应该做的

1. **及时回复** - 飞书消息 30 分钟内回复
2. **主动汇报** - 重要节点主动汇报
3. **简洁明了** - 汇报内容结构化
4. **问题预警** - 遇到问题提前告知

### ❌ 不应该做的

1. **沉默太久** - 超过 2 小时无反馈
2. **过度汇报** - 小事频繁打扰
3. **模糊描述** - "有点问题"这种描述
4. **隐瞒问题** - 遇到困难不及时说
